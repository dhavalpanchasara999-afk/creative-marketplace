========================================================================
             DIGIVAULT PREMIUM DIGITAL MARKETPLACE - DEPLOYMENT GUIDE
========================================================================

This workspace contains a dual-architecture deployment:
1. Immediate Single-Page-App (SPA) Offline Testing (HTML, CSS, JS with LocalStorage DB).
2. Production-grade Serverless API backend (Next.js, MongoDB/Mongoose, JWT, and Stripe).

Follow the steps below to run, customize, and launch your platform live.

---

1. IMMEDIATE OFFLINE SPA TESTING (NO INSTALLATION REQUIRED)
-----------------------------------------------------------
You can run and test all components (Auth, Cart, Gateway, Locker, Admin CRUD, and CMS Customizer) immediately in any web browser:
*   Simply open/double-click "index.html" in your browser.
*   All data operations, sales metrics, product additions/removals, and password sign-ups will be persisted instantly using your browser's local sandbox storage (LocalStorage).

To test administrative privileges:
*   Sign In using the pre-seeded admin profile:
    - Email: admin@digivault.in
    - Password: admin123
*   Once signed in, click the glowing purple "Visual Customizer" floating button or route to the "Admin Portal" in the navigation bar to access the CMS text editors and the CRUD datatable inventories.

---

2. PRODUCTION NEXT.JS FULL-STACK DEPLOYMENT
-------------------------------------------
To run this in a production-ready cloud server connected to a live database:

Step A: Install Project Dependencies
*   Open your workspace directory in a terminal and execute:
    npm install

Step B: Configure Live Environment Parameters
*   Create a ".env.local" file in the root folder and add the following keys:
    
    # MongoDB Connection Pool URL
    MONGODB_URI=mongodb+srv://<username>:<password>@cluster0.mongodb.net/digivault?retryWrites=true&w=majority
    
    # Secret Key for JWT Token Cryptography
    JWT_SECRET=digivault_ultra_secure_server_jwt_secret_token_123
    
    # Stripe Billing API Keys (Optional)
    STRIPE_SECRET_KEY=sk_live_...
    STRIPE_WEBHOOK_SECRET=whsec_...

Step C: Launch Local Server Execution
*   To start the hot-reloading Next.js dev server on port 3000:
    npm run dev
*   Open http://localhost:3000 in your browser to view the active platform.

Step D: Build Production Bundles
*   To compile highly optimized static assets:
    npm run build
*   To run the optimized server:
    npm run start

---

3. DEPLOYING TO VERCEL IN 1 CLICK
---------------------------------
Because Vercel is the creator of Next.js, deploying this full-stack application takes under 2 minutes:

1. Push your workspace folder to a private or public GitHub Repository.
2. Log in to Vercel at https://vercel.com
3. Click "Add New" > "Project" and import your DigiVault repository.
4. Under "Environment Variables", copy-paste the keys from your ".env.local" file (MONGODB_URI, JWT_SECRET, etc.).
5. Click "Deploy" - Vercel will build and launch your full-stack marketplace globally on a secure, SSL-encrypted .vercel.app subdomain!

---

4. CONNECTING CUSTOM DOMAINS (digivault.in)
-------------------------------------------
1. Inside your Vercel Project Dashboard, navigate to: Settings > Domains.
2. Enter your custom domain name (e.g. digivault.in) and click "Add".
3. Head to your Domain Name Registrar (e.g. Hostinger, GoDaddy, Namecheap) and add the target DNS records:
   *   For Root Domains: Add an A record pointing to: 76.76.21.21
   *   For Subdomains: Add a CNAME record pointing to: cname.vercel-dns.com
4. Vercel will automatically generate and renew SSL safety certificates for your domain instantly.

---
